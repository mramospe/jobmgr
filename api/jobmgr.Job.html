Job¶

class jobmgr.Job(executable, opts, odir, kill_event=None, registry=None)[source]¶

Bases: jobmgr.jobs.JobBase

Represent a step on a generation process.

Parameters:
  • name (str) – name of the job.
  • executable (str) – application/version to run.
  • opts (list(str)) – options to be passed to the executable.
  • odir (str) – where to create the output directory.
  • kill_event (threading.Event) – event associated to a possible “kill” signal. By default an event is constructed.
  • registry (JobRegistry or None) – instance to register the object. If “None”, the object will be registered in the main JobManager instance.
Variables:
  • executable – Command to be executed.
  • jid – Job ID, determined by the subdirectories in the output path.

peek(name='stdout', editor=None)[source]¶

Open the “stdout” or “stderr” file in terminal mode.

Parameters:
  • name (str) – log-file to open (“stdout” or “stderr”). The default is “stdout”.
  • editor – editor to open. If None is provided, then it will try to use “emacs” or “vi” to open the file. A RuntimeError is raised if none of them exist in the system, or a ValueError if the provided editor is not accesible.
Raises:
  • ValueError – if name is not “stdout” or “stderr”, or if the provided editor is not accesible.
  • RuntimeError – if “editor = None” and neither “emacs” nor “vi” are accesible.
