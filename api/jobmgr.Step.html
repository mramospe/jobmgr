Step¶

class jobmgr.Step(name, executable, opts, parent, data_regex=None, data_builder=None)[source]¶

Bases: jobmgr.jobs.Job

Represent a step on a generation process.

Parameters:
  • name (str) – name of the step.
  • executable (str) – application/version to run.
  • opts (list(str)) – options to be passed to the executable.
  • parent (SteppedJob) – parent job, inheriting from SteppedJob.
  • data_regex (str) – regex representing the output data to send to the next step.
  • data_builder (function) – function to define the way how the data is passed to the executable. It must take a list of strings (paths to the data files), and return a merged string. The default behaviour is to return list to be executed as: <executable> file1 file2 … but one can also define the string so the argument is explicitely specified, like: <executable> –files file1 file2 …
Raises:

RuntimeError – if the name used for this step is already being used in another step.
Variables:
  • executable – Command to be executed. Input data is added when the process just before execution (once it is defined).
  • name – Name of the step.
  • data_builder – Function that modifies the input data to parse it to the executable.
  • jid – Job ID, determined by the subdirectories in the output path.
